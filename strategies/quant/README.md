# dev-quant

## Installation

```bash
git clone https://github.com/yourusername/dev-quant.git
cd dev-quant
pip install -r requirements.txt
```

## Project files

- `data_fetcher.py`: Fetches market data in the portfolio management's format. 
- `config.py`: Project configuration definitions and settings.
- `feature_registry.py`: Feature registration and management utilities.
- `strategies.py`: Strategy implementations used by the project.
- `interface.py`: Interface definitions and abstraction points for the system.
- `main.py`: Entry point for the project. Orchestrates the workflow.
- `requirements.txt`: Python dependency list.

## Data flow

1. Raw market or input data is passed through `interface.input_adapter` to normalize and adapt the dataset.
2. `main.py` selects a strategy from `strategies.STRATEGY_MAP` and loads its required feature definitions.
3. The required features are generated by `feature_registry.FeatureRegistry` methods such as `compute_rsi`, `compute_moving_averages`, and others.
4. The chosen strategy's `alpha_func` uses the assembled features to compute alpha scores.
5. Those alpha scores are converted into discrete trading signals via thresholding and discretization.
6. Final signals are formatted by `interface.output_adapter` before being returned.
